# District Prioritization Strategy

## 核心原则

不按学区规模排序，而按 **搜索需求 × 竞争强度 × 数据维护成本** 排序。

不做"散点明星"，而做"**州集群**"——一个州一次做透。

---

## 为什么做州集群？

```
Florida
    Miami-Dade        ← 主 anchor
    Broward           ← 内链互指
    Palm Beach        ← 内链互指
    Orange            ← 内链互指
    Hillsborough      ← 内链互指
```

好处：
- 州页面 `/florida` 内容丰富，有排名能力
- Related Districts 可以自然互链，内链权重集中
- Google 容易识别主题权威性：这是一个专做 School Calendar 的平台
- 后续新增学区不需要调整信息架构

---

## Priority Score

```
Priority Score =
  Search Volume      × 0.5
+ Population         × 0.2
+ Brand Recognition  × 0.1
+ Data Quality       × 0.1
+ Internal Link Value × 0.1
```

Tier 3（50–300 学区）扩张时，按此打分排序。

## 第三梯队 — 规模化（50–300 个）

按 Priority Score 排序扩张，不再手动筛选。

每个学区必须满足：
- 可获取的官方日历数据（网页或 PDF）
- 至少 2025-2026 学年数据
- 对应州已有至少 1 个学区（利用现有内链结构）
